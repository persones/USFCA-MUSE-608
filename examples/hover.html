How does this work?

Notice this line:

.hover-container:hover .hover-target {

This is a complicated CSS selector. :hover is a pseudo-class, which is, in short, a class that is automtically assigned by the browser. In this case, it is when an element is hovered..

When two CSS selectros are written on after the other, without space between them, the rule applies to all elements that qualify to both selectors. So .hover-container:hover means "all elements that are of the class .hover-container and are currently hovered".

When two CSS selectros are written on after the other with a space between them, that is called a "descendant combinator", which means that the rule applies to elements that qualify to the second selector if they are within an element that qualify to the first selector. Example:

.parent-class p

will apply to p elements that are with in an element of ths class parent-class

So this line:

.hover-container:hover .hover-target

means: "all elements of the class .hover-target that are within an element of the class .hover-container which is currently hovered

If we now look at the rest of the CSS code, we see that the .hover-target class has a display: none property, but if the .hover-container, which contains is, is hovered, then the display becomes block, and the element appears. It has a position: absoluteproperty, so it shows exactly where we tell it to, without interfering with the rest of the page. Notice that you have to give the contianer, in this case .hover-container, a position: relative propery.
